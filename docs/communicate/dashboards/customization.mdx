## URL and Domain Configuration

### Default Dashboard URLs
Every dashboard starts with a unique, randomly generated URL:

```text
# Default URL format
https://{random-id}.checkly-dashboards.com

# Example
https://a1b2c3d4-e5f6-7890-abcd-ef1234567890.checkly-dashboards.com
```

### Custom Subdomain Setup
Create memorable, branded dashboard URLs under the Checkly domain:

<Steps>
<Step title="Choose Subdomain">
Select a unique subdomain that represents your organization
</Step>

<Step title="Configure Subdomain">
Set the custom subdomain in dashboard settings
</Step>

<Step title="Verify Availability">
Checkly verifies the subdomain is available across all dashboards
</Step>

<Step title="Update URLs">
Your dashboard becomes available at the new subdomain immediately
</Step>
</Steps>

```text
# Custom subdomain format
https://{your-subdomain}.checkly-dashboards.com

# Examples
https://acme-status.checkly-dashboards.com
https://api-health.checkly-dashboards.com
https://prod-monitoring.checkly-dashboards.com
```

<Note>
Custom subdomains must be unique across all Checkly users. Choose something specific to your organization to avoid conflicts.
</Note>

### Full Custom Domain Configuration
Use your own domain for maximum branding control and professional appearance:

<Steps>
<Step title="Domain Configuration">
Add your custom domain in the dashboard settings
</Step>

<Step title="DNS Setup">
Create the required DNS records at your domain provider
</Step>

<Step title="Domain Verification">
Complete domain verification if required by Checkly
</Step>

<Step title="SSL Certificate">
Automatic SSL certificate provisioning and renewal
</Step>

<Step title="Go Live">
Dashboard becomes accessible at your custom domain
</Step>
</Steps>

#### Required DNS Records

```text
# CNAME Record (required)
Type: CNAME
Name: status (or your chosen subdomain)
Value: checkly-dashboards.com
TTL: 3600 (or your domain provider's default)

# TXT Record (if domain verification required)
Type: TXT
Name: _checkly-domain-verification.status
Value: [Verification code provided by Checkly]
TTL: 3600 (or your domain provider's default)
```

#### Domain Configuration Examples

```yaml
# Example custom domain setups
Customer Status Page:
  Domain: status.acme-corp.com
  Purpose: External customer communication
  SSL: Automatic via Let's Encrypt
  
Internal Operations:
  Domain: monitoring.internal.company.com
  Purpose: Internal team visibility
  Access: Password-protected
  
Partner Integration:
  Domain: api-status.partner-portal.com
  Purpose: Partner API status communication
  Branding: Partner-focused styling
```

<Warning>
DNS changes can take up to 24-48 hours to propagate globally. Plan custom domain setup accordingly if you have time-sensitive launch requirements.
</Warning>

## Responsive Design Considerations

### Multi-Device Optimization
Ensure your dashboard works well across different screen sizes and devices:

```css
/* Responsive design example */
/* Mobile optimization */
@media (max-width: 768px) {
  .dashboard-header {
    padding: 1rem;
    text-align: center;
  }
  
  .check-card {
    margin-bottom: 0.5rem;
    padding: 1rem;
  }
  
  .dashboard-main {
    padding: 1rem;
  }
}

/* Tablet optimization */
@media (min-width: 769px) and (max-width: 1024px) {
  .check-card {
    flex: 1 1 calc(50% - 1rem);
  }
}

/* Desktop optimization */
@media (min-width: 1025px) {
  .check-card {
    flex: 1 1 calc(33.333% - 1rem);
  }
}

/* Large display optimization */
@media (min-width: 1440px) {
  .dashboard-container {
    max-width: 1400px;
    margin: 0 auto;
  }
}
```

## Customization Best Practices

### Design Guidelines
- **Consistent branding**: Maintain visual consistency with your organization's brand
- **Accessibility**: Ensure sufficient color contrast and readable typography
- **Performance**: Optimize images and CSS for fast loading times
- **Responsiveness**: Test dashboard appearance on various screen sizes

### User Experience Optimization
- **Clear hierarchy**: Use visual hierarchy to prioritize important information
- **Intuitive navigation**: Make it easy for users to understand dashboard content
- **Professional appearance**: Maintain credibility with polished design
- **Loading performance**: Ensure fast loading times for all users

<Note>
Remember that dashboard customization should enhance communication effectiveness, not just visual appeal. Always prioritize clarity and usability in your design decisions.
</Note>

<Warning>
Extensive CSS customization can sometimes interfere with dashboard functionality. Test thoroughly after making styling changes, especially for interactive elements and responsive behavior.
</Warning>
