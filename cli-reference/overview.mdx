The Checkly CLI enables monitoring as code, allowing you to create, test, and deploy synthetic monitoring checks directly from your codebase. Write checks in JavaScript/TypeScript, test them locally, and deploy them to Checkly's global monitoring infrastructure.

## Quick Start

### Installation Options

**Using npx (Recommended):**
```bash
npx checkly --help
```
Always uses the latest version without global installation.

**Global Installation:**
```bash
npm install -g checkly
checkly --help
```

**Project Installation:**
```bash
npm install --save-dev checkly
npx checkly --help
```

### Basic Workflow

```bash
# 1. Authenticate
npx checkly login

# 2. Test checks locally  
npx checkly test

# 3. Deploy to Checkly
npx checkly deploy
```

## Core Commands

### Authentication
Manage your Checkly account access:

- [`checkly login`](./login) - Authenticate with your account
- [`checkly logout`](./logout) - Sign out and remove tokens  
- [`checkly whoami`](./whoami) - Display current account info
- [`checkly switch`](./switch) - Switch between accounts

### Development & Testing
Develop and validate your monitoring setup:

- [`checkly test`](./test) - Run checks locally for testing
- [`checkly runtimes`](./runtimes) - List available runtime environments
- [`checkly sync-playwright`](./sync-playwright) - Sync Playwright configuration

### Deployment & Management
Deploy and manage your monitoring infrastructure:

- [`checkly deploy`](./deploy) - Deploy checks to your account
- [`checkly destroy`](./destroy) - Remove all project resources
- [`checkly trigger`](./trigger) - Run deployed checks on-demand

### Environment & Import
Manage variables and import existing resources:

- [`checkly env`](./env) - Manage global environment variables
- [`checkly import`](./import) - Import existing resources into CLI

## Global Options

All commands support common options for configuration and output control. See the complete list in [Global Options](./global-options).

**Most commonly used:**
- `--config` - Specify configuration file
- `--env` - Pass environment variables  
- `--verbose` - Show detailed output
- `--force` - Skip confirmation prompts

## Configuration

### Project Setup
Create a `checkly.config.ts` in your project root:

```typescript
import { defineConfig } from 'checkly'

export default defineConfig({
  projectName: 'My Monitoring Project',
  logicalId: 'my-project',
  repoUrl: 'https://github.com/user/repo',
  checks: {
    frequency: 300, // 5 minutes
    locations: ['us-east-1', 'eu-west-1'],
    tags: ['production'],
  },
  cli: {
    runLocation: 'eu-west-1',
    reporters: ['list'],
  },
})
```

### Multiple Environments
Use different configs for different environments:

```bash
# Staging environment
npx checkly test --config checkly.staging.config.ts
npx checkly deploy --config checkly.staging.config.ts

# Production environment  
npx checkly deploy --config checkly.production.config.ts
```

## Common Workflows

### New Project Setup
```bash
# Initialize project
npm create checkly@latest

# Authenticate
npx checkly login

# Test locally
npx checkly test

# Deploy
npx checkly deploy
```

### Daily Development
```bash
# Test changes locally
npx checkly test

# Deploy with preview
npx checkly deploy --preview

# Deploy after review
npx checkly deploy
```

### CI/CD Pipeline
```bash
# Authenticate with service account
npx checkly login

# Test monitoring
npx checkly test --reporter ci

# Deploy without confirmation
npx checkly deploy --force

# Clean up
npx checkly logout
```

### Import Existing Setup
```bash
# Import existing checks
npx checkly import

# Review generated code
npx checkly import apply

# Commit to CLI management
npx checkly import commit
```
